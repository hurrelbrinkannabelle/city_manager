I used ChatGPT(OpenAI) as an AI assistant to help generate parts of the filter functionality required in Phase 1 of the project.

*For the int parse_condition(const char *input, char *field, char *op, char *value) function I asked :I have a string in the format field:operator:value (for example severity:>=:2). Generate a C function that splits this string into three parts: field, operator, and value taking in account my structure and wanting to use it in a filter function.(int parse_condition(const char *input, char *field, char *op, char *value));
-it generated: 
int parse_condition(const char *input, char *field, char *op, char *value) {
    char *token = strtok((char*)input, ":");

    if (token != NULL) strcpy(field, token);

    token = strtok(NULL, ":");
    if (token != NULL) strcpy(op, token);

    token = strtok(NULL, ":");
    if (token != NULL) strcpy(value, token);

    return 1;
}
After trying it, it was obvious it didn t take care of some cases like: No validation if tokens are missing, always returns 1 (even if parsing fails),no buffer safety. So I worked in solving these issues and function looks like this now:

int parse_condition(const char *input, char *field, char *op, char *value){
    char buffer[256];
    strncpy(buffer, input, sizeof(buffer) - 1);
    buffer[sizeof(buffer) - 1] = '\0';

    char *f = strtok(buffer, ":");
    char *o = strtok(NULL, ":");
    char *v = strtok(NULL, ":");

    if (!f || !o || !v) return 0;
    strcpy(field, f);
    strcpy(op, o);
    strcpy(value, v);

    return 1;
}

*For the  int match_condition(Report *r, const char *field, const char *op, const char *value) function I asked AI :Now generate a function that checks if a Report matches field,operator,value, which returns 1 if the record satisfies the condition and 0 otherwise (int match_condition(Report *r, const char *field, const char *op, const char *value)).=>
int match_condition(Report *r, const char *field, const char *op, const char *value) {
    if (strcmp(field, "severity") == 0) {
        int v = atoi(value);
        if (strcmp(op, "==") == 0) return r->severity == v;
        if (strcmp(op, ">") == 0) return r->severity > v;
    }

    if (strcmp(field, "category") == 0) {
        if (strcmp(op, "==") == 0)
            return strcmp(r->category, value) == 0;
    }

    return 0;
}
It had issues looking at project requirment like: Missing many operators (!=, <=, >=), no support for inspector or timestamp, no validation of invalid fields, partial and incomplete logic.
So I worked on the issues and function ended up looking like this:

int match_condition(Report *r, const char *field, const char *op, const char *value){
    if (strcmp(field, "severity") == 0) {
        int v = atoi(value);

        if (strcmp(op, "==") == 0) return r->severity == v;
        if (strcmp(op, "!=") == 0) return r->severity != v;
        if (strcmp(op, ">")  == 0) return r->severity > v;
        if (strcmp(op, "<")  == 0) return r->severity < v;
        if (strcmp(op, ">=") == 0) return r->severity >= v;
        if (strcmp(op, "<=") == 0) return r->severity <= v;
    }

    if (strcmp(field, "timestamp") == 0) {
        long v = atol(value);

        if (strcmp(op, "==") == 0) return r->timestamp == v;
        if (strcmp(op, "!=") == 0) return r->timestamp != v;
        if (strcmp(op, ">")  == 0) return r->timestamp > v;
        if (strcmp(op, "<")  == 0) return r->timestamp < v;
        if (strcmp(op, ">=") == 0) return r->timestamp >= v;
        if (strcmp(op, "<=") == 0) return r->timestamp <= v;
    }

    if (strcmp(field, "category") == 0) {
        if (strcmp(op, "==") == 0) return strcmp(r->category, value) == 0;
        if (strcmp(op, "!=") == 0) return strcmp(r->category, value) != 0;
    }

    if (strcmp(field, "inspector") == 0) {
        if (strcmp(op, "==") == 0) return strcmp(r->inspector, value) == 0;
        if (strcmp(op, "!=") == 0) return strcmp(r->inspector, value) != 0;
    }

    return 0;
}

What I Learned?

Through this AI-assisted implementation, I learned several important concepts:

1. AI is not always correct or complete
The initial solutions generated by the AI were useful as a starting point, but they were incomplete and sometimes unsafe.
For example:
	parse_condition modified the original input string, which is unsafe when working with argv;
	match_condition did not implement all required operators or fields;
This showed me that AI-generated code must always be carefully reviewed and tested.
2. Safe string handling in C
I learned the importance of:
Copying strings before using strtok() because it modifies the input;
Using strncpy() instead of strcpy() when dealing with buffers;
Always ensuring null-termination of strings.
This helped me avoid bugs and undefined behavior.
3. Input validation is essential
The AI version did not validate input properly.
I learned to:
	Check if tokens exist after splitting (NULL checks)
	Return failure (0) when parsing is invalid
	Avoid assuming user input is always correct
3. Importance of understanding your code
This exercise showed me that:
	•	Writing code is not enough — I must be able to explain it
	•	AI can assist, but I must fully understand and adapt the result
	•	Debugging and improving generated code is where real learning happens
